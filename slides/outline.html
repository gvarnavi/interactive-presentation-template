Markdown preformatted code blocks with the python language identifier are automatically converted to runnable and editable code blocks thanks to react-py.

For example, in a Markdown docs file:

```python
import numpy as np

print("Hello, World!")
print(np.arange(10))
```

Becomes:

import numpy as np

print("Hello, World!")
print(np.arange(10))

We have included numpy in our default install configuration, but if you want any additional packages you can specify them using a comma-separated list, per code-block:

```python packages="scipy python-cowsay"
from scipy import special
from cowsay import cowsay

num = 27
res = special.cbrt(num)
message = f"The cube root of {num} is {res}"
print(cowsay(message))
```

You'll notice the examples before used print() statements to pipe outputs to stdout. This works reasonably well, but is restricted to textual outputs.

import os
import base64
from io import BytesIO

# Set this _before_ importing matplotlib
os.environ['MPLBACKEND'] = 'AGG'

import matplotlib.pyplot as plt

# Patch
def ensure_matplotlib_patch():
  _old_show = plt.show

  def show():
    buf = BytesIO()
    plt.savefig(buf, format='png')
    buf.seek(0)
	
    # Encode to a base64 str
    img = "<MatplotlibOutput>"
    img += 'data:image/png;base64,'
    img += base64.b64encode(buf.read()).decode('utf-8')
    img += "</MatplotlibOutput>"
    # Write to stdout
    print(img)
    plt.clf()

  plt.show = show

ensure_matplotlib_patch()

# Plot
fig, ax = plt.subplots()
ax.plot([1,3,2])
plt.show()

Above, we patch matplotlib's show() function to encode figures as a base64 png str to stdout, which we catch and render as an image using the MatplotlibCodeEditor component.
